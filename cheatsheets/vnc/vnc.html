Instalar VNC en Linux (RHEL)

En esta sección encontrarás información sobre cómo configurar un servidor VNC en Linux.

1. Actualizar el sistema e instalar el servidor TigerVNC

user@linux-server:~

# Actualizar el sistema e instalar el servidor TigerVNC

sudo dnf upgrade -y

sudo dnf install tigervnc-server -y

2. Configuracion del usuario(s) en el servidor

user@linux-server:~

# Crear el directorio de configuración y otorgarle los privilegios

sudo mkdir ~/.vnc

sudo chmod 700 ~/.vnc

# Definir una nueva contraseña para la sesión vnc del usuario

vncpasswd

# En caso de que no se genere un archivo llamado passwd, ejecutamos el comando

vncpass /home/$USER/.vnc/passwd

# Otorgarle los privilegios al archivo

chmod 600 ~.vnc/passwd

3. Configuración del servicio en el servidor.

Para esta configuración, iniciaremos con la asignación de pantallas a cada usuario

user@linux-server:~

sudo vi /etc/tigervnc/vncserver.users

De tal forma que quede de la siguiente manera:

user@linux-server:~

:1=user1

:2=user2

Preferencias del usuario

En este archivo, se define la resolución de la pantalla para cada usuario.

user@linux-server:~

sudo vi ~/.vnc/config

La configuración recomendada es la siguiente:

user@linux-server:~

session=xfce

geometry=1920x1080

alwaysshared

Script de inicio

En este archivo se editan el script de inicio para cada usuario, como el escritorio que se desea utilizar.

user@linux-server:~

sudo vi ~/.vnc/xstartup

La configuración recomendada es la siguiente:

user@linux-server:~

#!/bin/bash

unset SESSION_MANAGER

unset DBUS_SESSION_BUS_ADDRESS

# Iniciar el entorno de escritorio XFCE

startxfce4 &

4. Inicio del servicio

Para iniciar el servicio, se debe ejecutar el siguiente comando:

user@linux-server:~

# Recargar la configuración del servicio

sudo systemctl daemon-reload

# Habilitar el servicio

sudo systemctl enable vncserver@:1.service

# Iniciar el servicio

sudo systemctl start vncserver@:1.service

# Verificar el estado del servicio

sudo systemctl status vncserver@:1.service

5. Probar el servicio

Para probar el servicio, se debe ejecutar el siguiente comando desde la pc donde nos queremos conectar al servidor:

user@linux-server:~

# Verificar el estado del servicio

ssh -L 5901:localhost:5901 user@linux-server

Así es como terminan estos pasos. En caso de dudas puedes consultar la documentación oficial de TigerVNC o buscar en internet.
